> В базе данных MS SQL Server есть продукты и категории. Одному продукту может соответствовать много категорий, в одной категории может быть много продуктов. Напишите SQL запрос для выбора всех пар «Имя продукта – Имя категории». Если у продукта нет категорий, то его имя все равно должно выводиться.

Здесь используется связь многие ко многим. В этом случае задачу можно решить с помощью таблицы связи (продукт - категория).

Например, у нас есть следующие таблицы:

- **Products (Продукты)**: содержит продукты.
  - ProductId (Идентификатор продукта)
  - ProductName (Имя продукта)

- **Categories (Категории)**: содержит категории.
  - СategoryId (Идентификатор категории)
  - CategoryName (Имя категории)

- **ProductCategories (Продукты_Категории)**: таблица связи между продуктами и категориями.
  - ProductId (Идентификатор продукта)
  - CategoryId (Идентификатор категории)

Для запроса используем LEFT JOIN чтобы отобразить все продукты, включая те, у которых нет категории:

```sql
SELECT p.ProductName, c.CategoryName
FROM Products p
LEFT JOIN ProductCategories pc ON p.ProductId = pc.ProductId
LEFT JOIN Categories c ON pc.CategoryId = c.CategoryId
ORDER BY p.ProductName, c.CategoryName;
```
